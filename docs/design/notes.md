# UX Reference Notes

Use these notes to capture what each image should inspire. The goal is to preserve the product lesson, not copy the exact UI.

## References

### `ux-references/alltrails.jpeg`

- Inspiration: conversational support can feel lightweight, direct, and embedded in a consumer product experience.
- Potential takeaway: keep the chat entry point easy to discover without making it the only path through the app.

### `ux-references/alltrails_greetings.jpeg`

- Inspiration: greeting copy can set expectations before the user asks a question.
- Potential takeaway: use short, warm opening text that explains what the chatbot can help with without sounding like documentation.

### `ux-references/lyft_touch_button.jpeg`

- Inspiration: support flows can use clear touch targets and guided actions before or during chat.
- Potential takeaway: when the bot asks for an upload or escalation confirmation, make the action obvious and easy to tap.

## Guidelines

- Use these as directional UX references only.
- Do not copy brand-specific visuals, wording, layouts, icons, or proprietary interaction details.
- Convert useful ideas into original patterns that fit this chatbot project.
